COMSTAR SRP Target Release Notes

 Now that the COMSTAR SRP Target has integrated into SXCE 116, the following page is no longer needed. We will keep it around temporarily in case anyone needs to refer to it.

Hardware Requirements

The software has been tested using the ConnectX HCA using the "hermon" driver. (The InfiniHost HCA using the "tavor" driver and the InfiniHost III card using the tavor-compatibility mode (i.e. using the "tavor" driver) may also work but are not supported.)

Software requirements

The software is delivered in the form of packages that can be installed onto an existing version of OpenSolaris. While it is recommended that each release of these packages be installed into the same base version of the OpenSolaris operating system that the packages were built from, an OS upgrade is only strictly required when there has been an incompatible change (i.e. a Flag Day) that affects SRPT or IBDMA. In particular, snv107 represented a Flag Day for all Infiniband consumers. Our SRPT releases a001 and later should not be installed in OS releases earlier than snv_107.

Building From Sources

The sources can be downloaded from the above table. To build the SRP target:

  • download the source for the ON consolidation for the referenced recommended build
  • download the source tarball
  • untar the source tarball to add the SRP source files to your tree
  • run a nightly build.
  • The resulting packages will be in packages/ARCH/nightly or packages/ARCH/nightly_nd
  • (For more information on downloading and building the ON consolidation, see the onnv_build page.)

Download

 You can download binaries and sources from each release from the Binary and Source Downloads table on the main SRP page, which is also where you can see detailed hardware and software requirements.

Installation

 See the SRPT configuration and installation page for complete instructions on how to download and install these packages.

Changes in Release a_006, March 3, 2009

 Starting with a_006, the SRP target is controlled by SMF using the FMRI

svc:/system/ibsrp/target:default

 With previous builds, the COMSTAR targets for each HCA were created immediately as soon as the srpt driver was installed. With a_006, you need to enable the srpt service before any STMF targets will be created:

svcadm enable ibsrp/target

 To stop all SRP target services and remove the targets:

svcadm disable ibsrp/target

 CRs included in this release:

  • 6798265 Add an SMF service for SRP target
  • 6811539 Infinite loop in srpt_ch_task_mgmt_abort()

 Minimum OS Release: snv_107
