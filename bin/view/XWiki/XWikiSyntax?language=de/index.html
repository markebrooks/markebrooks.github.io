Beginnend mit XWiki Enterprise Version 1.7 haben wir eine neue Wiki-Syntax eingeführt. Die bisherige Syntax nennen wir XWiki Syntax v1.0, die neue Syntax logischerweise XWiki Syntax v2.0. Die Hauptgründe für die Einführung der neuen Syntax waren:

  • Behebung der Einschränkungen und Unklarheiten der XWiki Syntax v1.0 (welche durch Radeox, die zugrunde liegende Rendering-Engine, und TWiki inspiriert wurde).
    • Keine optimalen Steuerzeichen. Zum Beispiel war das Steuerzeichen für fett ein einzelner Stern. Dies hat Probleme verursacht, wenn Anwender in Texten Sterne verwendet haben, da dieser Text irrtümlich als fett interpretiert wurde. Daher haben wir die Syntax so verändert, dass überall mindestens doppelte Steuerzeichen notwendig sind.
    • Mehrdeutigkeiten. Zum Beispiel gab es eine Unklarheit zwischen einem fett dargestellten Element und einer Aufzählung. Die Auflösung dieser Mehrdeutigkeiten war erforderlich, um den WYSIWYG-Editor so umzuschreiben, dass er deterministisch werden konnte.
  • Annäherung an die Creole 1.0-Syntax, welche zum Standard für die Wiki-Syntax geworden ist. Zusätzlich hat sich die Creole-Gemeinschaft vor der Entscheidung über die Steuerzeichen die Zeit genommen, alle vorhandenen Wiki-Syntax-Ausprägungen zu analysieren. Die getroffenen Entscheidungen sind also sehr gut.

Zusätzlich zu diesen 2 XWiki-Syntaxversionen haben wir auch unsere zugrunde liegende Rendering-Engine (bisher Radeox) zugunsten der eigenen Engine ersetzt. Diese stellt eine Obermenge von  Wikimodel und Doxia (und möglicherweise anderen in der Zukunft) dar. Dies hat uns erlaubt, andere Schreibweisen im Wiki zu bereitzustellen: MediaWiki, Confluence, JSPWiki, Creole, TWiki und mehr.

Text Formatierung

In XWiki Syntax 2.0 gegenüber XWiki Syntax 1.0 neu:

  • Leerzeichen direkt nach Syntax Symbolen sind erlaubt (zum Beispiel war in XWiki Syntax 1.0 dieser Text nicht * fett *).
  • Verdopplung des Steuerzeichens gegenüber dem einfachen Steuerzeichen in XWiki Syntax 1.0, so dass es unwahrscheinlich wird, dass der Anwender es versehentlich im Text verwendet. 
  • Die Fähigkeit den Text über mehrere Zeilen zu schreiben (war nicht der Fall unter XWiki Syntax 1.0).

XWiki Syntax 1.0 Link Spezifikation

Das Format für einen Link ist (alias[|&gt;])+(link)(@interWikiAlias)+([|&gt;]target)

  • alias: Ein optionaler Text, der dem Anwender statt des Links angezeigt wird. Beispiel: "Meine Seite".
  • link: der vollständige Link mit folgender Syntax: (virtualWikiAlias:)+(space.)+(reference)(?queryString)+(#anchor)+
  • Bemerkung: entweder der link oder der alias müssen angegeben werden.
    • virtualWikiAlias: Ein optionaler String mit dem Namen einer virtuellen Wiki. Der Link zeigt auf eine Page in dieser virtuellen Wiki. Beispiel: "mywiki".
    • space: Ein optionaler Wiki Space Name. Falls kein Space angegeben ist, wird der aktuelle verwendet. Beispiel: "Main".
    • reference: Die Link Referenz. Das kann entweder eine URI in dem Formatprotocol:path(Beispiel: "http://xwiki.org", "mailto:john@smith.com") oder der Name einer Wiki Page (Beispiel: "WebHome").
    • queryString: Ein optionaler Such-String für die Parameter Spezifikation, der in der gerenderten URL benutzt wird. Beispiel: "mydata1=5&amp;mydata2=Hello".
    • anchor: Ein optionaler Anker Name, der auf einen Anker im referenzierten Link verweist. Beachten Sie, dass in XWiki für Überschriften automatisch Anker erstellt werden. Beispiel: "TableOfContentAnchor".
  • interWikiAlias: Ein optionaler Inter Wiki Alias, wie er in der InterWiki Map hinterlegt ist (siehe Admin Guide). Beispiel: "wikipedia"
  • target: Ein optionaler String übereinstimmend zum HTML target Attribut für ein HTML A Link Element. Dieses Element wird benutzt, wenn der Link gerendert wird. Die Voreinstellung öffnet den Link in der aktuellen Seite. Beispiel: "_self", "_blank"

XWiki Syntax 2.0 Link Spezifikation

The full format of a link is (label>>)?(link)(||parameters)*

  • label: Ein optionaler Text, der Wiki Syntax enthalten darf.
  • link: die Link Adresse - wohin gesprungen wird, wenn der Link angeklickt wird. Sie hat das gleiche Format wie in XWiki 1.0 Syntax und sie kann ebenfalls zwei Typen von URIs enthalten: mailto: und attach: um zu einer Mail-Adresse oder einem Anhang zu verweisen.
  • parameters: eine optionale Liste von Parametern, die zum Link an den Browser übergeben werden.

Makros

Es gibt zwei Arten von Makros in der XWiki Syntax 1.0:

  • Velocity Makros (Aufruf über die #macroname(param1 ... paramN) Syntax)
  • Radeox Makros (Aufruf über die {macroname:param1=value1|...|paramN=valueN} Syntax)

Es gibt ebenso zwei Arten von Makros in der XWiki Syntax 2.0:

  • Velocity Makros (Aufruf über die #macroname(param1 ... paramN) Syntax in Velocity Makros)
  • XWiki Makros (Aufruf über die{{macroname param1="value1" ... paramN="valueN"}} Syntax)
Die Radeox Makros können in XWiki Syntax 2.0 nicht verwendet werden. Deshalb wurden sie als XWiki Makros erstellt.
Für eine vollständige Liste von verfügbaren Makros siehe die Makro Referenz Seite.

HTML

Bei der XWiki Syntax 2.0 muss HTML oder XHTML über das HTML Makro eingegeben werden, während es in der XWiki Syntax 1.0 möglich ist, HTML direkt in den Text einzugeben.
XWiki Syntax 1.0XWiki Syntax 2.0Ergebnis
<b>fett</b>{{html}}<b>fett</b>{{/html}}fett
Standardmäßig versteht das HTML Makro in XWiki Syntax 2.0 keine Wiki Syntax (dies betrifft auch andere Makros, weil sie ebenfalls zur Wiki Syntax zählen). Um die Wiki Syntax zu erlauben verwenden Sie {{html wiki="true"}}.

Velocity/Groovy Skripte

In der XWiki Syntax 1.0 war es möglich, Velocity Skripte direkt überall auf der Seite einzugeben. Dies führte zu Problemen, zum Beispiel, wenn der Anwender unabsichtlich Velocity Inhalte eingab. Für Seiten ohne Velocity Code war dies Performance hungrig. Es gab mehrere andere technische Beschränkungen. Folglich müssen Velocity Skripte in XWiki Syntax 2.0 unter Benutzung des velocity Makros eingegeben werden. Das gleiche gilt für die Eingabe von Groovy Skripten.
FeatureXWiki Syntax 1.0XWiki Syntax 2.0
Velocity Skript
#set ($var = "whatever")

{{velocity}}
#set ($var = "whatever")
{{/velocity}}
Groovy Skript
<%
def var = "whatever"
%>

{{groovy}}
def var = "whatever"
{{/groovy}}

Parameter

Mit der XWiki Syntax 2.0 ist es möglich, den verschiedenen Syntax-Elementen und auch Textblöcken Parameter mitzugeben. Das wird zum Beispiel verwendet, um den XHTML "style" zu setzen. Sie können jedes beliebige Schlüssel/Wert Parameter-Paar eintragen. Der XHTML Renderer wird diese Parameter als XHTML Attribute an das jeweils zu Grunde liegenden XHTML Syntax-Element weiterreichen. In der XWiki Syntax 1.0 gab es keine Syntax für die Weitergabe von Parametern. Der einzige Weg die Formatierung zu erreichen, war das direkte Schreiben von XHTML in den Beitrag, wie in der Tabelle unten beschrieben.

XWiki Syntax 1.0XWiki Syntax 2.0erstelltes XHTML
<h1 class="myClass" style="myStyle" id="myId">Überschrift</h1>

(% class="myClass" style="myStyle" id="myId" %)
= Überschrift =

<h1 class="myClass" style="myStyle" id="myId">Überschrift</h1>

Andere Syntaxen

Die folgenden anderen Syntaxen werden ab XWiki Enterprise 1.6 unterstützt:

  • JSPWiki
  • MediaWiki
  • Confluence
  • TWiki
  • Creole 1.0
  • XHTML 1.0
Die Implementierung für diese Syntaxen ist noch nicht beendet. Zum Beispiel arbeitet die Unterstützung für Links noch nicht vollständig. Wir müssen auch definieren, ob wir die ursprünglichen Syntaxen erweitern wollen, um XWiki spezifische Feature zu unterstützen, wie die Fähigkeit sich mit einem sub-wiki zu verbinden.
