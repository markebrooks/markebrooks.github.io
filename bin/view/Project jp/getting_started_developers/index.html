はじめての開発者のための FAQ


はじめての開発者のための FAQ
ここには、ほかの FAQ よりも、OpenSolaris プロジェクトのより技術的な質問と回答が掲載されています。

  • まず何を行なったらよいですか。
  • ソースコードについては、何を行うことが許可されていますか。
  • OpenSolaris のソースコードとバイナリはどこで入手できますか。
  • ビルド環境の設定、適切なツールの入手、および実際のソースビルドに関する情報はどこにありますか。
  • ソースが更新される頻度はどのくらいですか。
  • コードをビルドしました。次に何を行なったらよいですか。
  • どうすればコミュニティーに参加できますか。
  • OpenSolaris ソースコードの使用中にバグを発見しました。報告するにはどうすればよいですか。
  • セキュリティー関連のバグは、ほかのバグとは扱いが異なりますか。
  • OpenSolaris コードベースの変更を提供するにはどうすればよいですか。
  • 何に取り組むべきかについてのアドバイスはありますか。
    まず何を行なったらよいですか。
    OpenSolaris のソースコードとバイナリを、ソースブラウザを使って閲覧し、ダウンロードすることが最初のステップです。
    OpenSolaris のソースコードとバイナリはどこで入手できますか。
    ダウンロードのページに、ソースをビルドするためのソフトウェア要件と手順が含まれています。
    OpenSolaris のソースコードとバイナリについては、何を行うことが許可されていますか。
    OpenSolaris ソースコードは、自由に閲覧、変更、ビルド、使用、および再配布できます。各バイナリのダウンロードには固有のライセンスがあり、それぞれ異なる権利と制限事項が含まれています。ソースコードとバイナリに関するライセンス条項については、ライセンスのページを参照してください。
    ソースはどこで入手できますか。
    ビルド可能な OpenSolaris コードの最新バージョンは、ダウンロードのページからダウンロードできます。
    ビルド環境の設定、適切なツールの入手、および実際のソース構築に関する情報はどこにありますか。
    ビルドするには、まずリリースノートを参照してください。リリースノートは、ダウンロードしたソースおよびダウンロードのページにあります。リリースノートには、ソースをビルドするためのソフトウェア要件など、特定リリースの OpenSolaris ソースコードに関して知っておく必要のあるすべての情報が記載されています。
    開発者リファレンスマニュアルを参照すれば、必要なツールの詳細情報など、ソースのビルドに関するほとんどの疑問に対して答えが見つかるはずです。このマニュアルには、最新の OpenSolaris バイナリを使ったシステムの更新方法や、開発者として知っている必要のあるあらゆる情報も記載されています。
    ソースが更新される頻度はどのくらいですか。
    ソースは約 2 週間ごとに更新されます。今後は、更新の頻度を上げることを計画しています。
    コードをビルドしました。次に何を行なったらよいですか。
    最初に、興味のあるコミュニティーに参加してください。どのような仕事がすでに着手されているかについて、および最適な貢献を行う方法について理解するのに役立ちます。OpenSolaris プロジェクトに貢献するには、次のように多くの方法があります。
  • コミュニティーおよびディスカッションに参加する
  • バグを発見および報告する
  • ドキュメントまたはソースコードを提供する
  • プロジェクトをより成功させる方法についてのフィードバックを提供する
    どうすればコミュニティーに参加できますか。
    コミュニティーは、特定の技術分野の情報およびディスカッションを集中させる主要な手段です。将来は、Web サイトの機能によって各コミュニティーが複数のプロジェクトをホストできるようになり、コミュニティーがすべてのプロジェクトに共通する情報を提供するとともに、プロジェクト固有の情報を提供するインフラストラクチャーを提供できるようになる予定です。
    コミュニティーポータルおよび Website FAQ を参照すれば、コミュニティーに参加して Web サイトを使用するために必要なすべての情報を入手できるはずです。
    OpenSolaris ソースコードの使用中にバグを発見しました。報告するにはどうすればよいですか。
    最初に、バグデータベースを検索して、その問題がすでに報告されているかどうかを確認してください。このデータベースはだれでも検索できますが、バグを提出できるのは登録済みの (サイトにサインインした) ユーザーだけです。
    現在のところ、OpenSolaris コードベースに含まれている技術に対するバグの検索と記録のみが可能です。ソースコードを利用できるようにするうえで、Solaris OS のその他の技術を含めることができるようにバグ報告機能を拡張する予定です。opensolaris.org の次回の重要な更新では、既存のバグを更新するためのサポートが追加されます。
    セキュリティー関連のバグは、ほかのバグとは扱いが異なりますか。
    明白な理由から、OpenSolaris コードに含まれるセキュリティー脆弱性 (またはその可能性) に関する情報については、注意深くありたいと考えています。脆弱性を発見したと思われる場合は、 Sun Security Coordination Team に警告してください。
    OpenSolaris コードベースへの変更を提供するにはどうすればよいですか。
    OpenSolaris プロジェクトの改善ページには、貢献するコードに関連する開発フェーズの概要、およびより詳細なドキュメントへのリンクが記載されています。
    何に取り組むべきかについてのアドバイスはありますか。
    独自のアイデアやコミュニティーディスカッションから学んだ内容に加えて、キーワード oss-bite-size を使ってバグデータベースを検索すると、Solaris 技術チームが小規模で自己完結型の変更と識別したバグを見つけることができます。これらは、コードに精通するのに役立ちます。
    バグの修正や機能に関するアイデアがある場合、または取り組んでみたい既存のバグが見つかった場合は、request-sponsor@opensolaris.org のエイリアスにメールを送信して、OpenSolaris ソースベースに変更を反映させる作業を開始してください。
