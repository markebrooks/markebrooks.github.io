Welcome

Welcome to the X Keyboard Layout project page.

The purpose of this project is to continuously improve and extend the keyboard layout support for OpenSolaris in cooperation with Open Source Communities and OpenSolaris Users from different countries and for all different languages supported in OpenSolaris.

In this page, all information related with XKB will be gathered together. This project contains a subset of minor projects already known by many OpenSolaris Users as the "OpenSolaris Standard Keyboard Layouts" and others.

Location of all keyboard data in G11n repositories, how to download the source code, how to build Xkeyboard layout data, how to contribute with new keyboard layouts, how to build and set a new keyboard layout, etc... all this information will be provided in successive steps within this project.

Overview

Currently, OpenSolaris as a XKB-based system uses a modified version of Xkeyboard-config v1.6. Xkeyboard-config is a regularly maintained open source database and it is part of the FreeDesktop project.

The modified version of Xkeyboard-config v1.6 for OpenSolaris contains additional support for Sun Keyboards but otherwise they are almost identical, it provides support for more than 400 keyboard layouts variants for more than 70 countries and regions around the world and for more than 180 keyboard models.

One of the goals of this project is to increase the keyboard layout coverage for OpenSolaris.

OpenSolaris standards for Keyboard Layouts

This project is already well known by many users of OpenSolaris. The project began several months ago and its main objective was to define the "OpenSolaris Standard Keyboard Layouts" in cooperation with Users and Open Source Communities from different countries for all languages supported in OpenSolaris.

The successful outcome of this subproject can be described as the creation of a new Keyboard Layout Variant for a given language. This new variant must be approved by a significant number of community group members as well as by users of the given Keyboard Layout because it will be used as the default keyboard layout for the given language in OpenSolaris.

Below you can find a list of the Keyboard Layouts that Community members have contributed so far:

(Albanian, Brazilian, Canadian_bilingual, Czech, Estonian, Georgian, Greek, Kazakh, Latvian, Lithuanian, Polish, Romanian, Slovak and Ukrainian)

If you consider that some of the Keyboard Layouts listed above is not correct and it should be changed or updated for any of these languages; you can present a proposal of a different Keyboard Layout and present it to the corresponding community group for approval. (more details...)

For the following keyboard Layouts there are also already defined its standards for OpenSolaris:

(Arabic, Belgian, Danish, Dutch, Finnish, French, German, Italian, Japanese, Korean, Norwegian, Portuguese, Russian, Spanish, Swedish, Swiss/French, Swiss/German, Turkish Q, UK, and US)

A complete set of characters and symbols for these Keyboard Layouts can be found here: Sun USB Type 6 keyboard

There are still many Keyboard Layout Variants that can be defined and you can contribute with them.

Getting the source code

The source code containing all Keyboard Layout data for OpenSolaris can be downloaded from hg.opensolairs.org. Everyone can browse or download the code anonymously however if you want to contribute with bugfixes you will need an account with additional permissions (please contact project's leaders for details)
You can pull the source from the G11n Mercurial repository; the following command will create your own clone of the complete G11n repository:

hg clone ssh://anon@hg.opensolaris.org/hg/nv-g11n/g11n g11n

This workspace contains not only keyboard data but it contains also other g11n related stuff as fonts, locale data, etc. The Keyboard Data for Xorg can be found in this directory: src/xorg/keytables.

Getting involved

Building the workspace

If you want to make some changes for the keyboard layout data in your local machine, usually you don't need to build the whole g11n workspace. You can just backup your current files and then replace them with the new modified files and after restarting the Xserver, all new files should be correctly loaded. 

General information on the g11n Nevada project can be found here.

Reporting new bugs

Bugs for the X Keyboard Layout data in OpenSolaris are mostly tracked in Bugzilla – OpenSolaris Defect Tracking. To report a new bug against X Keyboard Layout data in bugzilla; select OpenSolaris based binary distribution and i18n-l10n component, then complete the form describing the issue and the steps to reproduce it.

A list of open bugs related with X keyboard layouts can be found here

Issues and discussions related to X keyboard layouts are discussed at i18n-discuss discussion mailing list
