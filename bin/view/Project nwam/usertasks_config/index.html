User Tasks for NWAM Configuration

User Tasks for NWAM Configuration GUI

 This lists the configuration user tasks that can be accomplished by the first release of NWAM. Also see the user tasks for status notification. Some tasks are marked "(ADVANCED)" to indicate if the task is considered for a more network savvy user.

 For ease of reference, "connections" refers to the IP interfaces and links.

User tasks for first release

  • Network Status
    • See what connections are connected, which are not
    • See strength of wireless signal if connected
  • Network configuration
    • Add network connection (ADVANCED)
      • Tunnel
        • Give vanity name
        • 3 types: 6to4, IPv4, IPv6
      • Ethernet NIC
        • Specify vanity name
        • Specify Device name
        • Max Packet Size (Standard/Custom)
      • Wireless NIC
        • Specify vanity name
        • Specify Device name
    • Edit network connection properties
      • View/Edit IP address designation
        • View IP address
        • IPv4: Manual assignment,Multiple (ADVANCED), DHCP, None
        • IPv6: Manual assignment, Advanced (ADVANCED), DHCP, None
        • For DHCP only: allow user to specify options such as timeout and fallback address
      • Allow user to persist settings for the connection
    • Remove a network connection
    • Enable a connection
    • Disable a connection
    • Enable/disable a connection based on what else is connected
    • Edit Conditions
    • See graph view of network connections
    • See list view of network connections, their types, enabled/disabled/conditionally enabled status
    • Wireless Network Connection
      • View available wireless networks and their strength
      • Sort by name and strength
      • Specify order of preference for wireless networks
      • Allow user to enter and store WEP passwords
      • Allow user to differentiate between base stations by BSSID (ADVANCED)
  • Environment
    • Create a new Environment
    • Edit Environment
      • Edit Proxies for (HTTP, HTTPS, Gopher, SOCKS, FTP)
      • Edit Name Service (to some extent..)
      • Enable/disable network services. (ADVANCED)
      • Add network services specific to user (ADVANCED)
      • Specify Environment activation conditions
      • Edit Routing & Forwarding options (ADVANCED)
    • Switch Environments (manually or automatically via conditions set by user)
    • Remove Environment
    • Duplicate Environment

User tasks post-first release

  • All of the above plus,
  • Add/edit network connection expanded to IPMP Group, VLAN, Link Aggregation (ADVANCED)
  • Edit/switch in different network configurations (NCP) (ADVANCED)
  • Inheritance in environments (ADVANCED)
