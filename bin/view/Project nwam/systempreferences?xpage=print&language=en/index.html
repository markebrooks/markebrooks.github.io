System Preferences

 This is where all network configuration is done. There is a surprising amount of functionality crammed into here.

 The network related settings are located in a row of four icons:

 The first of these has settings for Apple's proprietary ".mac" service. The third is settings related to their QuickTime multimedia framework (a surprising number of which seem network related).

Network Preferences Pane

 The second, labeled Network, is where most "network-address" related settings are configured. The network settings are collected into groups called "Locations". From the online help:

A network location is a group of settings for your network ports (your computer's Ethernet port, modem port, or AirPort Card, for example).

If you often change network settings, you may find switching between locations quicker and easier than repeatedly modifying individual groups of settings.
For example, you might want to make a location for "home," where you connect to the Internet with a telephone modem, and a location for "office," where you connect to your company network and the Internet using Ethernet or AirPort.

 Each Location, in turn, has a set of Port Configurations, which have a definite order. Each Port Configuration, in turn, has settings for that particular "port" (link) including how to get an address, DNS settings, proxy settings, etc.

 Note that Locations are shared by all users. This might be a bit of a surprise since, say, the ".Mac" settings are per-user.

 The current location is only changed manually. Port configurations, on the other hand, are automatically changed based on their order and the presence/absence of network "signal". Changing a Location seems to reset all applications to use the new network settings (e.g. IM client logs off and then re logs in all accounts), but changing the Port Configuration does not.

Network Status

 When this "Network" preference pane is first opened, it shows a summary of the current status of the ports for the current Location.

 In this case, this indicates that the default Location (called Automatic) has two potentially active ports: AirPort (wireless) and the internal modem. As you'll see in a moment, there are actually five ports defined as part of this Location, however three of them have no current potential for network connection, so are not shown here. If an ethernet cable were plugged in, it would then show up in this list.

 Note that the "Configure..." button at the bottom will switch the display to showing the configuration settings for that particular Port Configuration. the "Disconnect..." button will start the "Internet Connect" application where one can disconnect etc.

Common Network Interface

 Some parts of this are common across all portions of the Network Preference Pane.

 The "Assist me[sic]..." button displays a sheet asking what kind of assistance you want. "Diagnostics..." leads to the Network Diagnostics application, while "Assistant..." leads to the Network Setup Assistant.

 The Location menu shows you the current locations as well as providing commands for changing these. In this screenshot, it shows two locations, the default "Automatic" and a newly created one called "Example Location"

 Choosing to create a new Location simply displays a sheet which asks for a new name. The resulting Location has a Port Configuration for each port and no settings in any of the Port Configurations.

 Edit Location presents a sheet which lists the various locations. Double clicking on a location name is the same as clicking the Rename button and allows the Location's name to be changed. Duplicate and Delete do what you would expect.

 The "Show" menu has three sections. The first displays the network status for the currently active Location. Perhaps this is a bug, and perhaps it is a feature, but the "currently active" one is the one that last had its settings applied. Thus, if you switch the current location from the menu, and then display the "Network Status" information without first clicking "Apply Now", the status will show the status for the Location that currently is in effect on the system.

 The next several items show each of the Port Configurations for the Location listed in the menu above in their preferred order. Choosing one of those allows you to edit/configure that particular Port Configuration. The last item gives a list of the Port Configurations for the Location listed in the menu above.

Network Port Configuration

 The Network Port Configuration section displays a list of all the Port Configurations for the Location listed at the top of the window.

The order of these ports is significant. In the next illustration, if there is network signal from both the Internal Modem and from the Built-in Ethernet, apps will use the Internal Modem Port Configuration if they try to make a network connection. These can be re-ordered using drag and drop:

 Port Configurations can also be made inactive, which means that the system will ignore those ports even if there is network signal there. This is done simply by unchecking the checkbox in the list (the Port Configuration remains in this list, but it disappears from the Show menu, above).

 It is possible to create a new Port Configuration. It isn't at all clear to me at the moment what some of this means or when it would be used. I assume this gets into much more advanced networking needs.

 When you first click on the "New..." button, a sheet appears asking for the name of the port, and what port that name should be associated with. In this example, I've typed "New Port", though the default is the classic "Untitled".

 If you expand the Port menu, you get not only a list of the ports we already know about, but two special ones, a "6 to 4" (I assume this is IPv6 to IPv4) and a Ling Aggregate.

 The last is most interesting, because when that is chosen, the sheet expands to display which ports may be aggregated. Why this lists only the two is not clear to me. Note that the underlying names (en0 and en1) have begun to poke their ugly noses into the end-user interface.

 Proceeding with the creation of a Link Aggregation port, this alert is then shown.

 I doubt it is very clear what is going on. As soon as one clicks OK, the ports in the list that you'd chosen to aggregate are disabled (but, not deleted), and the aggregated port is then listed. It is worth noting that the "Rename" button becomes "Edit..." when you have selected a Link Aggregate port, and that redisplays the sheet shown above so that you can change both the name and change which ports are in the aggregate.

 The "Rename" button (double clicking on a Port Configuration name does the same thing) allows you to edit the name of the Port Configuration.

 Deleting a Port Configuration does one of two things. If this is a Port Configuration that was created by a user, then the whole port configuration is deleted and removed from the list. However, if it is one of the ones in the list when it was created, then the "Delete" button effectively deletes the Port configuration and creates a new (with original name), empty of settings, and inactive (unchecked) Port Configuration for the associated underlying link/port.

 Duplicate makes a copy of the specified Port Configuration. I'm not sure what problem this solves.

Sharing Preferences Pane

 The "sharing" icon in this set provides access to configuration information having to do with how the host computer is accessed from other computers. IMO, the category name and icon are not particularly evocative of what is found here. On the other hand, these are settings that ordinary folks aren't likely to change.

 Note that the hostname is specified at the top of this preferences pane.

 In the services tab, remote-access services can be turned on and turned off.

 Note that selecting (as opposed to enabling or disabling) any service my reveal other controls on the right that allow some specialized control of that service.

 The firewall tab allows the system's firewall to be turned on and off, and provides an easy way to allow certain common things through. Turning on a service will automatically enable the appropriate hole(s) in the firewall.

 You can edit the settings for some of these items (ElectricSheep is a freeware app which added itself to the list of things allowed through the firewall when it was installed)

 You can also manually add your own.

 Interestingly, some common port names come predefined. If you choose a predefined one, you may not edit the port fields.

 Finally, clicking on the Advanced button brings up some interesting settings
