Wiki Evaluation Form

Wiki Evaluation Form

 This is the format for evaluating wiki technologies. It is intentionally quite simple. There are three section - software information, dependency information and requirements fulfillment, each with explanatory notes. Evaluation forms should follow this layout and be posted to website-discuss.

Software

 Include the name and version of the software being evaluated, with references to the website. For example;

 Software name: MyWiki
 Software version: 1.0.1
 Website: www.mywiki.org

Dependencies

 List any dependencies the software has and include the name and version of the required software used for the evaluation. For example;

 Required software: MySQL
 Version: 5.1.4
 Required software: Apache Tomcat
 Version: 6.0.14

Requirements Fulfillment

 Refer to the Wiki Requirements page. Evaluate the software based on each of the requirements using a 'yes/no' response. Include references and explanatory notes if possible. For example;

 E0 The software must be free and open source
 Yes. GPL licensed. See [insert link]

 E1 The software must have a page history
 Yes. The software has a page history, and supports page histories for translations. It does not support revision diffs however. See [insert link]

 ...

 C13 The software should have an active community
 Yes. See mailing list activity at [insert link].

 C14 The software should have pluggable content storage
 No. Flat file storage only.

 ...

 O31 Virtual server support
 Yes. See [insert link].

 O32 Localized page listing
 No

 etc.
