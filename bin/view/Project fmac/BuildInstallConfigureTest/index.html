How to Build, Install, Configure, and Test FMAC

How to Build, Install, Configure, and Test FMAC

The following steps describe how to build, install, configure, and test an FMAC system.

  1. Build FMAC.

Follow the instructions in the OpenSolaris ON Build Cheatsheet.

  1. Install SXCE97.

It's possible to install the FMAC BFU archives on a separate boot environment using Live Upgrade that will allow you to revert back to a working environment should the BFU'd image become unstable or unbootable (brick state). These instructions utilize Live Upgrade but one could BFU directly on the target SXCE97-based system as well.

a. Install the SXCE97 software.

  • Select "Solaris Express"
  • Select option 3 "Solaris Interactive Text (Desktop session)"
  • Perform normal install stuff
  • When prompted for "Select the file system to use for your Solaris installation" select ZFS and install the "Entire Distribution plus OEM support"

After installation, verify the system is configured and running properly

b. Download and Install the b97 onbld package

c. Create a new boot environment on which to BFU.

        # lucreate -n fmac_bfu

d. Activate the new image.

        # luactivate fmac_bfu

e. Reboot to start the new image (only use shutdown or init).

        # init 6

f. You should now be running in the BFU boot environment.

  1. Log in in as root and BFU.

a. Setup the correct BFU environment by using the bldenv command, and perform the BFU:

        # bldenv -d /net/system_name/builds/john/opensolaris.sh
        # bfu /net/system_name/builds/john/fmac-gate/archives/uname -p/nightly

Or perform a manual BFU by using the following manual methods:

        # FASTFS=/opt/onbld/bin/uname -p/fastfs; export FASTFS
        # BFULD=/opt/onbld/bin/uname -p/bfuld; export BFULD
        # GZIPBIN=/usr/bin/gzip; export GZIPBIN
        # EXTRACT_HOSTID=/opt/onbld/bin/uname -p/extract_hostid; export EXTRACT_HOSTID
        # PATH=/opt/onbld/bin:/opt/onbld/bin/uname -p:$PATH; export PATH
        # bfu /path_to_archives/archives/uname -p/nightly

b. Run Automatic Conflict Resolution (ACR).

        # acr

c. Reboot.

        # reboot

  1. Label the system.

a. When system comes back up, login as root and run the fmacsetup script.

        # /sbin/fmacsetup

b. Reboot.

        # reboot

  1. Verify FMAC functionality.

a. When the system comes back up, it should be in permissive mode. Login as root and verify that FMAC is functioning correctly:

        # /sbin/getenforce
        permissive
        # pcon
        101157: root:sysadm_r:sysadm_t:unclassified
        # newrole -r user_r -t user_t
        # pcon
        101161: root:user_r:user_t:unclassified
        uid=0(root) gid=0(root)
        # sum /etc/shadow
        29982 1 /etc/shadow
        # dmesg | grep avc | tail -1
        Oct 29 17:06:34 edeamtf genunix: [ID 702911 kern.notice] avc:  denied  { read } for
        scontext=root:user_r:user_t:unclassified tcontext=system_u:object_r:shadow_t:unclassified tclass=file
        pid=101172 comm=sum path=/etc/shadow
        # exit
        101209: root:sysadm_r:sysadm_t:unclassified
        # /sbin/setenforce enforcing
        # /sbin/getenforce
        enforcing
        # newrole -r user_r -t user_t
        # pcon
        101235: root:user_r:user_t:unclassified
        # id
        uid=0(root) gid=0(root)
        # sum /etc/shadow
        sum: /etc/shadow Permission denied
        # dmesg | grep avc | tail -1
        Oct 29 17:22:55 edeamtf genunix: [ID 702911 kern.notice] avc:  denied  { read } for
        scontext=root:user_r:user_t:unclassified tcontext=system_u:object_r:shadow_t:unclassified tclass=file
        pid=101237 comm=sum path=/etc/shadow

b. If you want to run with enforcing enabled by default, add the following line to /etc/system and reboot:

        # set fmac_enforcing=1

See the fmac(5) man page for additional information about FMAC boot flags and enforcement settings.

  1. Add a non-root admin user.

You might want to add a non-root admin user (sysadm) that maps to sysadm_u\:user_r\:user_t. The sysadm user will be able to login with the sysadm_u:user_r:user_t context then newrole -r sysadm_r -t sysadm_t as needed for FMAC related administrative tasks.

        # useradd -g staff -d /export/home/sysadm -m -c "System Administrator" -s /bin/sh sysadm
        # passwd sysadm some_passwd
        add context=sysadm_u\:user_r\:user_t to the user_attr entry for sysadm as mentioned above

  1. Perform subsequent BFUs steps.
