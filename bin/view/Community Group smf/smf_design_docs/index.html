SMF Project Documents

Discussion and ARC cases for SMF functionality. Comments and feedback are welcome.

RFE/FeatureStatusDescription
Relaxed Type Requirements for SMF Profiles PSARC 2010/222Integrated in snv_143.Do not require types to be specified in profiles when the type can be determined from existing repository data.
New Networking Types PSARC 2010/157Integrated in snv_141.Add new SCF_TYPE_NET_ADDR type for libscf values.
Site Profile Directories PSARC 2010/152Integrated in snv_141 (6945131).Allow site profiles to be delivered as fragments in a directory rather than one monolithic file.
Early Manifest Import PSARC 2010/013, EMI Design DocumentIntegrated in snv_137Allow for manifests to be imported early in the boot process before services are started or loaded from the repository.
SMF Notifications: PSARC 2009/617,PSARC 2009/618,PSARC 2009/619,PSARC 2010/278Integrated in snv_146.Allow configuration of email messages or SNMP traps when service state changes occur.  A transition to maintenance state for a service instance results in a defect diagnosis in FMA, with corresponding case lifecycle events.
Allow Property Modification in SMF profiles, PSARC 2009/371Integrated in snv_119.Allow properties to be modified in SMF profiles in addition to the existing ability to enable or disable instances.
method_context/credential simplification, PSARC 2009/093Integrated in snv_113.Allow partial method context definitions in the repository, and enable typical service/instance inheritance of method context and credential properties.
Improved maintenance diagnosis: PSARC 2008/730Integrated in snv_109.If maintenance is not requested by an administrator on a tty, have svcs -x display the fmri of the service which performed the svcadm mark maintenance.
Solaris host-based firewall (service-based firewall configuration): PSARC 2008/580Integrated in snv_109.Express system and service firewall configuration as a set of SMF properties. These are automatically translated by ipfilter into appropriate ipfilter rules, and can be easily manipulated by administrators or higher level firewall configuration tools (like the Visual Panels service firewall tool).
Template Extensions: PSARC 2008/350Integrated in snv_102.Extend SMF templates to provide a general mechanism for describing service configuration metadata.
SMF Property Value Ordering: PSARC 2008/432Integrated in snv_95.Enhance SMF property storage to make value lists ordered by default and in all cases.
svcs -xq: PSARC 2008/159ARC case approved.Permit svcs -x to give no output, but report in its exit code whether any services are in a problematic state.
Process Contract Decorations: PSARC 2008/046Integrated in snv_85.Add service FMRI to process contracts to increase observability.
Manifest Import Performance Improvements (6351623)Integrated in snv_84.Improve manifest-import time by importing to a tmpfs-backed repository. Pave the way for Early Manifest Import by allowing imports before the root filesystem is mounted read-write.
svccfg refresh: PSARC 2008/015Integrated in snv_82.Allow svccfg to update a running snapshot for a service instance on an alternate repository.
SMF Audit Integration: PSARC 2006/213Integrated in snv_81.Create BSM audit records for important SMF administrative events.
SMF Sensitive Properties: PSARC 2007/177Integrated in snv_74.Allow read-protection of sensitive application properties in the repository.
