Fast root ZFS boot SXCE Upgrade using lu tools

Open Solaris Express Community Edition and the more mainstream OpenSolaris 2008.xx builds differ significantly in package to package. One uses the new IPS system, the other doesn't currently. One uses the pkg method for upgrades based on dynamic "pull down" of IPS formed packages, the other uses lu or live upgrade. Here, we show just how easy it can be to upgrade the "full bodied" SXCE builds between versions.

What you'll need

  1. A currently installed open Solaris host. Ours was x86 snv b103
     2. A new DVD or DVD ISO image of the newer build. Ours was x86 snv b104
     3. A means to get the DVD ISO onto the hard disk, or transported over NFS to the target host. We used the iso directly on a spare zpool on the target host
     4. Enough space on the root ZFS boot volume to uncompress the full 3.32GB of OpenSolaris install twice over. You're creating two whole boot environments here, remember!

How to upgrade

 First, we need to prepare the "media" whether it be in the form of an ISO image fine or a DVD. It is by far easiest to simply use an ISO image file on the host itself.


bash 3.2# lofiadm -a /ztank/sol-nv-b104-x86-dvd.iso /dev/lofi/1
bash 3.2# mount -o ro -F hsfs -o ro /dev/lofi/1 /mnt
bash 3.2# pkgrm SUNWluu SUNWluzone SUNWlur SUNWlucfg
bash 3.2# pkgadd -d /mnt/Solaris_11/Product SUNWlucfg SUNWlur SUNWluu SUNWluzone

 So, you are probably wondering why we removed, then re-added the SUNWlu tools like this. Simply put, we needed to upgrade to the current versions to do this. It is recommended practice. You can also do this from Gnome desktop by double clicking on the DVD and then installing, by double clicking on the install.sh script in the root of the DVD media.

 Next, we need to actually MAKE our new boot environment.


bash 3.2# lucreate -c b103 -n b104

 Here, we create the new boot environment and designate the current with -c. It shows "current" being b103. We then make the "new (-n)" one as b104. The truth is, you can call them what you like!

 Next, we review our findings:


bash-3.2# lustatus
Boot Environment           Is       Active Active    Can    Copy      
Name                       Complete Now    On Reboot Delete Status    
~-------------------------- ~-------- ~------ ~--------- ~------ ~----------
b103                       yes      yes     yes       no    -         
b104                       no       no      no        no     -

 We want to get to a point where it's all on b104. Time to push the magic button.


bash 3.2# luupgrade -u -n b104 -s /mnt

 At this point, you'll see a LOT of data and text fly by. On a Q6600 intel quad system with 4GB of RAM and 7.2k RPM SATA hard disks, this entire process took 40 mins.

 Time to activate it!


bash 3.2# luactivate b104

 An important note:


**********************************************************************

In case of a failure while booting to the target BE, the following process
needs to be followed to fallback to the currently working boot environment:

1. Boot from Solaris failsafe or boot in single user mode from the Solaris
Install CD or Network.

2. Mount the Parent boot environment root slice to some directory (like
/mnt). You can use the following command to mount:

     mount -Fzfs /dev/dsk/c0d0s0 /mnt

3. Run  utility with out any arguments from the Parent boot
environment root slice, as shown below:

     /mnt/sbin/luactivate

4. luactivate, activates the previous working boot environment and
indicates the result.

5. Exit Single User mode and reboot the machine.

**********************************************************************

 Reboot with an init 6 always! After this, all done! Painless. The old version of the system can then be safely deleted at your leisure via ludelete.
