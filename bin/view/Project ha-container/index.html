Cluster Agent: OpenSolaris Container

Overview

The purpose of this project is to provide an umbrella for the ongoing development (like bug fixes and feature enhancements) of the already existing HA Container agent within Open HA Cluster.

Project Goals

  • Fix all open Bugs for the HA Container agent.
  • Implement accepted requests for enhancements (RFE) for the HA Container Agent.

Status

  • 10 March 2009 webrev send out for code review to fix CR 6815384 - get_ipmp_state() should use the parsable output from ipmpstat.
  • 06 March 2009 webrev send out for code review to fix CR 6814263 - sczsmf validate does fail if the locale used in global zone is not available in the non-global zone.
  • 04 March 2009 webrev send out for code review to fix CR 6812924 - get_ipmp_state() logic to configure LH for a zone with sczbt does not work with Clearview.
  • 15 January 2009 webrev send out for code review to introduce ipkg brand type support on OpenSolaris for project Colorado.
  • 23 July 2008 webrev send out for code review to fix CR 6707591 - scswitch causes error "svcs: Could not bind to repository server: repository server unavailable."
  • 10 Apr 2008 webrev send out for code review to implement RFE 6686683 - Support for the solaris9 brand type within the HA Container agent.
  • 02 Apr 2008 webrev send out for code review to fix CR 6662001, 6610950 and 6683336.
  • 25 Feb 2008 Umbrella Project for the HA Container agent got approved by the HA Clusters Community Group.

Documentation

  • Sun Cluster Data Service for Solaris Containers Guide
  • Section "Support for Solaris Zones on Sun Cluster Nodes Through Sun Cluster HA for Solaris Containers" within the Sun Cluster Concepts Guide for Solaris OS

Getting involved

  • Browse the source for the HA Container agent.
  • Talk with people or search the archives in the ha-clusters:discuss  discussion group
  • File a Bug or RFE using Category (suncluster) and Sub-category (ha-zones)
