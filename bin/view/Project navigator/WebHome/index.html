Navigator

Overview

The Navigator project provides a set of Sun Studio C/C++ projects enabling users to use the powerful Sun Studio IDE (or NetBeans IDE) with the ON sources. The Navigator project is aimed at students and others wanting to learn and explore the ON sources. The C/C++ projects enables users to browse and navigate the ON sources with all the power of an IDE like hyperlink navigation, class browser, call graphs, etc. The projects also enables users to seamlessly integrate third-party software like drivers with the ON sources. Building, running, and debugging ON components will be supported in a later release.

Setup

The steps to setup Sun Studio (or NetBeans) to work with the ON sources are simple:

  1. Use Sun Studio 12 update 1 (or newer) or NetBeans 6.5.1 (or newer). Both are available from the Package Manager. They are also available at http://developers.sun.com/sunstudio or http://www.netbeans.org. Important: update the IDE to latest version with Help->Check for Updates after installation.
  2. Download and unpack the ON sources and download and unpack the C/C++ projects in the same directory. It is important that the version of the C/C++ projects matches the build number of the OpenSolaris sources and the architecture of your system. Matching sources and C/C++ projects can be found here:

Working with the IDE and the C/C++ Projects

ON sources are located in the folders src and proto in the directory where you unpacked the sources and the projects. The C/C++ projects are in a folder called pomona. In pomona there are more than 1000 C/C++ projects matching commands, libraries, and the kernel in src. The project for the echo command is for instance in pomona/cmd/echo and libc is in pomona/lib/libc. The kernel is in pomona/uts. To open a project, simply start the IDE (run sunstudio or netbeans), click on Open Project... icon in the toolbar and use the project chooser to navigate to the project you want to open and then select OK. Projects have declared  dependencies so if you for instance open the echo project and select Open Required Projects from it's context menu, the libc project will also get opened. Once required projects are opened, you can quickly navigate from one project to another using the mouse and Ctrl+Left-Click navigation feature in the IDE. See here for more tips of how to use the IDE.

Important: some of the C/C++ projects are large and requires extra large memory heap. Start the IDE like sunstudio -J-Xmx1000M or netbeans netbeans -J-Xmx1000M to increase the heap.

Working with Third-Party Software

Integrating third-party software with the ON sources and the IDE requires you to create a C/C++ project around the third-party software and you declare the proper dependencies on the ON projcets. Once you have done that, editing features like code completion and hyperlink navigation work seamlessly between the projcets. The following example demonstrates how to integrate a device driver with the ON sources.

  1. Download the driver sources rf-2.4.0.tar.gz and unpack them somewhere. The driver is an open source Ethernet driver for Solaris.
  2. Start the IDE: sunstudio -J-Xmx1000M or netbeans netbeans -J-Xmx1000M
  3. Open the libc project. It's in pomona/lib/libc.
  4. Open the uts kernel project. It's in pomona/uts.
  5. Launch the C/C++ Project From Existing Code project wizard from New Project... icon in toolbar.
  6. In the first panel of the wizard, point the Existing Makefile to the Makefile of the driver in the directory where you unpacked the driver sources.
  7. Finish the wizard by accepting all other defaults. A project called rf-2.4.0_1 is created.
  8. Right-click rf-2.4.0_1 project and select Properties.
  9. Select Required Projects category and click on ... editor icon to select a dependent project. In the dialog that comes up, click Add button and point project chooser to uts project (pomona/uts). The kernel will be added as dependent project. Click Add again and select the libc project (pomona/lib/libc). libc will be added as dependent project
  10. Click OK a couple of times to acknowledge the changes

The driver project is now integrated with the kernel and libc. Navigation features like hyperlink navigation and code completion work seamlessly between the projects. Try to open the gem.c source file in the driver and go to line 992. Position the mouse over the ddi_dma_mem_free symbol and hit Ctrl+Left-click with the mouse. It will take you directly to the implementation of ddi_dma_mem_free in the kernel sources. Go back to line 992 in gem.c and enter a blank new line. Type the first few characters of a symbol, ddi_dma for instance, and hit Ctrl+Space. A code completion dialog comes up with suggestions of how to complete the symbol. For more tips of how to use the IDE, see here.
