ZFS Forensics

NOTE: The forensics project is no longer active on this website so information here may be out of date. Current Oracle Solaris 11 product documentation can be found here. Information about downloading Oracle Solaris 11 can be found here.

ZFS File System Forensics

 This page is dedicated to ZFS forensics, and will be used to gather tools and docs specific for this file system. UFS is by comparison well documented, especially in the book by Brian Carrier:  File System Forensic Analysis.

 There are not many tools specifically designed for forensic examination of ZFS file systems. However, there are some on-board tools for interpreting on-disk structures. Readers are refered to:

the ZFS project pages, especially the documentation of the on-disk structures, and

the ZFS GRUB code which gives a good overview of the ZFS code.

 ZFS has a debugging tool, zdb, which can be used to interpret on-disk structures in human-readable form. Although it does not give sector or byte offsets for the structures it finds, and is therefore hard reproduce, the output is very comprehensive when run with maximum verbosity:

zdb -vvvvvvv [zvol]

ZFS On-Disk Data Walk - Max Bruning

 From the proceedings of Open Solaris Developer Conference Prague 2008 this talk addresses on-disk ZFS structure. The presentation discusses techniques of walking trough the structures, working the pathname in order to get to the actual file data. At each step data structures are discussed

 Please find the proceedings here (starting at page 36) and the actual presentation here.
