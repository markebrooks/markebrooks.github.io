Roadmap 2009

Archive: 2011 | 2010 | 2009 | 2005-2008 | Consolidations | Website Announcements

WORK AREA JAN-MAR 2009 APR-JUN 2009 JUL-SEP 2009 OCT-DEC 2009
Website Development & Transition
  • New ("Auth") user
    Management Application integrated with XWiki and released for beta
    testing    . (12/08)
  • Content migration and translation tools developed. (3/09)
  • Content migration tests done internally. Building schedule for external testing. (3/09)
  • Migrated user data to Auth from multiple locations and posted documentation. (3/27/09)
  • SCM Console Phase 1 available (4/09) (Note: You must be logged in on the site to access the SCM console and see repositories for projects for which you are a Leader.)
  • Updated hub.os.org to
    XWiki v1.8.2 with new WYSIWYG editor. (5/09)
  • Began a series of open Website Transition conference calls (5 calls in total from April-Sept). See Website Community for dates, slides, notes. (4/24/09)
  • Published documents on Data Migration strategy and Roles & Collectives plans. (6/12/09)
  • Final beta update for Auth. (7/24/09)
  • Website Transition Phase 1: Promoted Auth to be live/lead user management database. (8/3/09)
  • Published documentation on Site Feature Mappings for tonic <-> XWiki. (8/6/09)
  • Updated Defect. (8/10/09)
  • Website Transition Phase 2: Began content migrations from Tonic to XWiki on hub.os.org (31 migrations over 3 months). (8/10/09)
  • Update roles in Auth. (10/5/09)
  • Website
    Transition Phase 2: Final transition to XWiki on
    hub.os.org.(10/26/09)
  • Collective moratorium ended. Began setting up new infrastructure on site. (10/28/09)
  • Deployed update to bugs.os.org. (11/11/09)
  • Updated Infrastructure Life Cycle Instructions w/ new processes for new site. (12/4/09)
  • Substantially updated two dozen common documents and wrote a new user guide and localization contribution model. (12/19/09)
Website Content Development
  • Feature spots: Starfish Project, Crossbow Project, SW Porters Group, Test Farm, OpenSolaris Bible, Tesla Project, OGB election, FOX Project.
    Sponsored links: VirtualBox 2.1, OpenStorage, CommunityOne NYC, MySQL, OpenOffice, TechDays, Application Competition, Storage Summit, OGB Election.
  • Information Architecture: Review of landing pages for Downloads, Groups, Projects, Docs, and About.
  • Develop content migration plan and schedule for template and checklist development to support the work.
  • Feature spots: Pro OpenSolaris, Open HA Cluster Summit, CommunityOne West Developer Conference, OpenSolaris 2009.06
  • Sponsored links: Java, Glassfish, CommunityOne SF, TechDays, NetBeans, VirtualBox.
  • Information Architecture: Cleanup landing pages and all other common pages.
  • Begin phase 1 testing of content migration for all common pages and provide template and checklist for first 100 collectives.
  • Feature spots: Boomer Project, Website Transition Phase 1, Documentation CG
  • Sponsored links: ISVs, OpenStorage, Free CD, TechDays
  • Information Architecture: Complete cleanup of 100 common pages and final testing of migration script.
  • Feature spots: Security Summit, Website Transition to XWiki, G11N CG
  • Sponsored links: Developer information, New documentation, OpenSolaris HowTo, Toshiba OpenSolaris Mini NB200, TechDays
  • Information Architecture: Common content migrated as part of transition to XWiki.
Open Development Infrastructure
  • bugs.opensolaris.org: Take ownership of bugs.opensolaris.org application and prepare to deploy on opensolaris.org servers (3/09)
  • Mercurial: Internal presentations about transitioning. Slides/instructions posted for Project Leaders. (2/09)
  • Mercurial: Upgrade to Mercurial 1.1.2. (3/09)
  • SCM Migration Project: Tools tasks
  • Webrev updated to handle bugs from both bugster and bugzilla. (3/09)
  • bugs.opensolaris.org: Deploy on os.org servers and use new search engine. (5/09)
  • defect.opensolaris.org: Take operational ownership of defect.os.org (5/09)
  • defect.opensolaris.org: upgrade to Bugzilla 3.4.1 (8/09)
  • bugs.opensolaris.org: stanadalone application/publish public comments field(11/09)
