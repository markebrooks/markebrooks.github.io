Procédure d'installation d'un cluster à nœud unique

NOTE: This page has not yet synced up with latest English page. Please refer to original English page for latest information.

Visionnez la séquence vidéo intitulée Procédure d'installation du logiciel Solaris Cluster Express à nœud unique qui vous décrit la procédure d'installation d'un cluster High Availability (HA) Solaris Sun à nœud unique !

Cette procédure fournit des instructions simplifiées permettant de créer un cluster sur un seul ordinateur, à utiliser à des fins de développement et de test d'un agent Open HA Cluster. Des informations supplémentaires sont disponibles dans la documentation Sun™ Cluster 3.2 : http://docs.sun.com/app/docs/prod/sun.cluster32.

Procédez comme suit pour créer votre cluster à nœud unique :

  1. Avant l'installation
  2. Installation du logiciel Solaris Express
  3. Installation du logiciel Solaris Cluster Express
  4. Établissement du cluster

Avant l'installation

Avant d'installer le logiciel, vérifiez que votre système présente la configuration suivante :

  • Votre matériel figure dans la liste de compatibilité matérielle à l'adresse suivante : http://www.sun.com/bigadmin/hcl/data/sx/.
  • Vous avez choisi les adresses IP du réseau public que vous comptez utiliser. Il vous faut une adresse IP pour le nœud de cluster et une autre pour chaque service de données à configurer.
  • Les logiciels Solaris Express Community Edition version 97 et Solaris Cluster Express sont disponibles pour l'installation. Effectuez l'une des opérations suivantes pour que le logiciel soit disponible :
    • Téléchargez le logiciel sur un serveur d'installation ou d'initialisation.
    • Gravez le logiciel téléchargé sur un DVD.

 Les instructions concernant le logiciel et son installation sont disponibles sur l'un des sites Web suivants :

  • Solaris Express Community Edition version 97 (enregistrement obligatoire)
  • Solaris Cluster Express (enregistrement obligatoire)

Retour au début

Installation du logiciel Solaris Express

 Cette procédure permet d'installer le logiciel Solaris Express Community Edition version 97.

Remarque : si votre système est déjà équipé de Solaris Express Community Edition version 97, le schéma de partitionnement doit répondre aux exigences de Solaris Cluster Express pour que la configuration du cluster réussisse.

  • Si les allocations et partitions nécessaires existent déjà (notamment la partition /globaldevices, il est inutile d'effectuer cette opération. Rendez-vous à la rubrique Installation du logiciel Solaris Cluster Express.
  • Si les allocations et partitions nécessaires n'existent pas, repartitionnez le disque système ou réinstallez le logiciel Solaris Express Community Edition version 97 en suivant les étapes de cette procédure.
  1. Suivez les instructions du site de téléchargement Solaris Express Community Edition http://opensolaris.org/os/downloads pour installer le SE sur votre système.
    Suivez ces instructions supplémentaires lors de l'installation.
    1. Effectuez les choix d'installation suivants :|=Invite|=Valeur
Groupe de logiciels SolarisPrise en charge OEM Entire Plus
PartitionsFormatage manuel
Mot de passe rootIndiquez le mot de passe root à utiliser.
Réinitialisation automatiqueNon
    1. Définissez les tailles de partitions et les noms de fichiers système suivants, s'ils ne sont pas déjà définis :|=Tranche|=Taille|=Nom du système de fichiers
0Moitié de l'espace disponible restant/

732 Mo(facultatif) pour l'utilisation du gestionnaire de volumes
    1. Une fois l'installation terminée et le système réinitialisé, connectez-vous en tant que superutilisateur.
    2. Restaurez l'accès externe à la communication RPC (appel de procédure à distance) et, éventuellement, à la console Web Sun Java™.
      Pendant l'installation de Solaris Express, un profil de réseau restreint est utilisé pour limiter l'accès externe à certains services du réseau qui affectent la fonctionnalité du cluster, notamment les services suivants :
      11* le service de communication RPC, requis pour la communication de cluster ;
      11* le service de la console Web Sun Java, requis pour l'utilisation de l'interface graphique de Cluster Manager.
      1. Exécutez les commandes suivantes pour restaurer l'accès externe à la communication RPC.
# **svccfg**
svc:> **select network/rpc/bind**
svc:/network/rpc/bind> **setprop config/local_only=false**
svc:/network/rpc/bind> **quit**
# **svcadm refresh network/rpc/bind:default**
# **svcprop network/rpc/bind:default | grep local_only**

La sortie de la dernière commande doit indiquer que la propriété local_only est maintenant définie sur false.

      1. (Facultatif) Exécutez les commandes suivantes pour restaurer l'accès externe à la console Web Sun Java.
# **svccfg**
svc:> **select system/webconsole**
svc:/system/webconsole> **setprop options/tcp_listen=true**
svc:/system/webconsole> **quit**
# **/usr/sbin/smcwebserver restart**
# **netstat -a | grep 6789**

La sortie de la dernière commande doit renvoyer une entrée pour 6789, qui représente le numéro de port utilisé lors de la connexion à la console Web Sun Java.

    1. Définissez les valeurs des variables  umask et PATH dans votre fichier d'initialisation d'utilisateur.
      1. Ouvrez votre fichier d'initialisation d'utilisateur pour l'éditer.
        Il s'agit en général du fichier .cshrc ou .profile dans votre répertoire de base.
      2. Assurez-vous que la variable PATH inclut les chemins suivants :/usr/bin
        /usr/cluster/bin
        /usr/sbin
        /usr/ccs/bin
      3. Vérifiez que la valeur de la variable umask est définie sur 22.
umask 022

Cette entrée définit les autorisations par défaut des fichiers récemment créés.

      1. Enregistrez et quittez votre fichier d'initialisation d'utilisateur.
    2. Mettez à jour le fichier /etc/inet/hosts avec le nom d'hôte et les adresses logiques publics du cluster.
      Attribuez une adresse IP de réseau public à chaque agent que vous souhaitez configurer.
127.0.0.1         localhost
//###.###.##.#//      //nodename//  loghost
//###.###.##.#//      //agentname//

Retour au début

Installation du logiciel Solaris Cluster Express Cette procédure permet d'installer les packages de la structure Solaris Cluster Express.

    1. Sur le système que vous installez, accédez au répertoire Solaris_sparc ou Solaris_x86 du logiciel Solaris Cluster Express téléchargé.
    2. Lancez le programme  installer .
# **./installer**
    1. Suivez les instructions s'affichant à l'écran pour installer les packages de la structure Solaris Cluster Express.
      Fournissez les informations suivantes lorsque vous y êtes invité.

Établissement du cluster

    1. Lancez l'utilitaire interactif scinstall.
# **/usr/cluster/bin/scinstall**

Le menu principal de scinstall s'affiche.

    1. Saisissez le nombre pour "Créer un nouveau cluster ou un nouveau nœud de cluster", puis appuyez sur Entrée.
      Le menu Nouveau cluster et Nœud de cluster s'affiche.
    2. Saisissez le nombre pour "Créer un nouveau cluster", puis appuyez sur Entrée.
      Le menu Standard ou Personnalisé apparaît.
    3. Saisissez le nombre pour "Standard", puis appuyez sur Entrée.
    4. Suivez les invites de menu pour fournir les informations suivantes :|=Composant|=Description|=Réponse
Nom du clusterQuel est le nom du cluster que vous souhaitez établir ?nom_cluster
Nœuds de clusterIndiquez le nom des autres nœuds.Appuyez sur Ctrl-d pour indiquer qu'il n'y a pas d'autre nœud dans le cluster.
ContrôleSouhaitez-vous interrompre l'installation en cas d'erreur de sccheck ?Non
Le cluster est établi une fois tous ses nœuds correctement réinitialisés. La sortie de l'installation de Solaris Cluster Express est consignée dans un fichier journal /var/cluster/logs/install/scinstall.log.N.
    1. Vérifiez que le cluster est établi.
# **cluster status**
=== Cluster Nodes ===
~--- Node Status ~---
Node Name                                       Status
~---------                                       ~------
//nodename//                                        Online
