Solaris Cluster Express - Versionshinweise

NOTE: This page has not yet synced up with latest English page. Please refer to original English page for latest information.

 Diese Versionshinweise enthalten Zusätze, Änderungen und Korrekturen an Informationen in der Dokumentation zu Sun Cluster 3.2, da diese Solaris™ Cluster Express und Solaris Cluster Express Geographic Edition mit dem Open HA Cluster-Projekt betreffen. Die Informationen im Dokumentationssatz zu Sun Cluster 3.2 haben für den Code in Solaris Cluster Express und Solaris Cluster Express Geographic Edition sowie dem Open HA Cluster-Projekt Gültigkeit, sofern hier nichts anderes angegeben ist.

  • Änderungen bei der Unterstützung
  • Neue Funktionen
  • Einschränkungen und Voraussetzungen
  • Bekannte Probleme und Umgehungen

Änderungen bei der Unterstützung

Solaris OS - Solaris Cluster Express kann jetzt unter Solaris Express Community Edition Build 97 ausgeführt werden.

Zurück zum Anfang

Neue Funktionen

Solaris Cluster Express

Die folgenden neuen Funktionen sind im Release von Solaris Cluster Express vorhanden:

Unterstützung für RAC in Zonen

 Dieses Projekt führt den Zonen-Cluster ein, welches ein virtueller Cluster ist, in dem der virtuelle Knoten eine Zone ist. Sie können auf einem einzigen Rechner mehrere Zonen-Cluster konfigurieren. Die Anwendungen in verschiedenen Zonen-Cluster sind komplett unabhängig. Zonen-Cluster bieten Sicherheitsisolation, Ressourcenmanagement auf Zonen-Cluster-Ebene und Anwendungsfehlerisolation. Wir bieten Single-Point-of-Administration von jedem Knoten aus sowie weitere praktische Funktionen, damit der Zonen-Cluster zu einem vereinfachten Cluster wird. Wir haben Oracle RAC 9i, Oracle RAC 10g und Oracle RAC 11g in verschiedenen Zonen gleichzeitig auf derselben Hardware unter Verwendung verschiedener Speichertopologien ausgeführt.

Ein- und Ausschalten der Sun Cluster-Datenabgrenzung (Fencing)

Sie können die Sun Cluster-Datenabgrenzung jetzt für bestimmte oder alle Geräte optional ein- und ausschalten.

Unterstützung zur Auswahl einer Partition oder eines Loopback-Dateitreiber (LOFI)-Geräts für den "Globale Geräte"-Namespace

Diese neue Funktion wandelt PxFS und "Globale Geräte"-Untersysteme in optionale Sun Cluster-Komponenten um, sodass Leistung, Bedienkomfort, Verfügbarkeit, Flexibilität und Wartung von Sun Cluster verbessert werden.

Behobene Programmierfehler für Agenten

 Eine aktualisierte Liste mit behobenen Programmierfehlern für Agenten ist verfügbar unter http://www.opensolaris.org/os/community/ha-clusters/ohac/Documentation/ohacds_changelog/.

Bekannte Probleme und Umgehungen

In den folgenden Abschnitten werden Umgebungen für nicht behobene Programmierfehler in Solaris Cluster Express und Solaris Cluster Express Geographic Edition beschrieben. Diese Umgehungen werden aus diesen Versionshinweisen gelöscht, wenn sie in verfügbaren aktualisierten Versionen des Codes von Solaris Cluster Express und Solaris Cluster Express Geographic Edition behoben sind.

[zh_CN]:Ergebnis der Prüfung von Systemvoraussetzungen ("system requirements checking") ist falsch (6495984)

Problembeschreibung: Wenn Sie Sun Cluster mit dem Java Enterprise System (JES)-Installationsprogramm installieren, meldet die Software, welche die Systemvoraussetzungen prüft, fälschlicherweise, dass der Swap-Speicher in den Gebietsschemata vereinfachtes und traditionelles Chinesisch 0 MB ist.

Umgehung: Ignorieren Sie diese Meldung.

 In diesen Gebietsschemata können Sie den folgenden Befehl ausführen, um den richtigen Swap-Speicher zu ermitteln:

# df -h | grep swap

Erforderliche Schritte, falls die Solaris-Option zur standardmäßigen Sicherung aktiviert ist - Dokumentation erforderlich (6558275)

Problembeschreibung: Während der Installation von Solaris Express deaktiviert die Einstellung eines eingeschränkten Neztwerksprofils den externen Zugriff auf zwei Netzwerkdienste, die von Solaris Cluster Express verwendet werden:

  • Der entfernte Prozeduraufruf (PRC)-Kommunikationsdienst, der für die Cluster-Kommunikation erforderlich ist
  • Der Sun Java™ Web Console-Dienst, der für die Verwendung der Cluster Manager-Benutzeroberfläche erforderlich ist

Umgehung: Stellen Sie für externen Zugriff wieder die RPC-Kommunikation und optional die Sun Java Web Console ein.

  1. Führen Sie die folgenden Befehle aus, um den externen Zugriff wieder auf RPC-Kommunikation einzustellen.
    # svccfg
     svc:> select network/rpc/bind
     svc:/network/rpc/bind> setprop config/local_only=false
     svc:/network/rpc/bind> quit
     # svcadm refresh network/rpc/bind:default
     # svcprop network/rpc/bind:default | grep local_only
     Die Ausgabe des letzten Befehls sollte zeigen, dass die Eigenschaft local_only jetzt auf false steht.
  2. (optional) Führen Sie die folgenden Befehle aus, um den externen Zugriff auf die Sun Java Web Console wiederherzustellen:
    # svccfg
     svc:> select system/webconsole
     svc:/system/webconsole> setprop options/tcp_listen=true
     svc:/system/webconsole> quit
     # /usr/sbin/smcwebserver restart
     # netstat -a | grep 6789
     Die Ausgabe des letzten Befehls sollte einen Eintrag für 6789 zurückgeben, was die Portnummer für die Verbindung mit Sun Java Web Console ist.

Mehrere DATS-Tests sind entweder nicht initiiert oder ungelöst (6571388)

Problembeschreibung: Einige Data Service Automated Test Suite (DATS)-Tests könnten als UNINITIATED (nicht initiiert) oder UNRESOLVED (ungelöst) fehlschlagen, weil eine abgefragte Ressourcengruppe noch nicht vollständig online ist. Das Problem geht auf einen Timing-Fehler zurück.

Umgehung: Derzeit ist keine Umgehung verfügbar.

Installation von SUNWdats-Paket schlägt fehl, falls es zuvor installiert und deinstalliert wurde (6574454)

Problembeschreibung: Ein Postinstallationsskript schlägt fehl, falls das SUNWdats-Paket zuvor installiert und deinstalliert wurde.

Umgehung: Importieren Sie die folgende XML-Datei manuell:

# svccfg import tcc-dtet.xml

rpc.metad Core Dump, wenn Festplatte zu Satz hinzugefügt wird (6579356)

Problembeschreibung: Dieses Problem in Solaris Cluster Express verhindert die Ausführung der zwei GEO-Szenarien geo_avs_mult_pgs_nonreboot und geo_avs_mult_pgs_reboot.

Umgehung: Derzeit ist keine Umgehung verfügbar.

neuen Schritt in scge-srdf-Failback-Verfahren einfügen (6717354)

Problembeschreibung: Nach dem kompletten Ausfall eines Originalprimärs funktionieren die Failback-Verfahren möglicherweise nicht, wenn Sie die Gerätegruppe EMC Symmetrix Remote Data Facility (SRDF) nicht zuerst in einen Failed Over-Zustand setzen.

Umgehung: Fügen Sie im Handbuch Sun Cluster Geographic Edition Data Replication Guide for EMC Symmetrix Remote Data Facility in den Abschnitten "How to Perform a Failback-Switchover on a System That Uses EMC Symmetrix Remote Data Facility Replication" und "How to Perform a Failback-Takeover on a System That Uses EMC Symmetrix Remote Data Facility Replication" vor dem Schritt, in dem Sie den Befehl geopg update protectiongroupname ausführen, die folgenden zwei neuen Schritte hinzu:

 Prüfen Sie in cluster-paris Rolle und Zustand der Gerätegruppe.

 # symrdf -g device-group query

 Wenn die Gerätegruppe die Rolle RDF1 besitzt und sich in einem Split-Zustand befindet, starten Sie ein Failover der SRDF-Gerätegruppe.

 # symrdf -g device-group failover

Aktualisierung von Failback-Switchover-Verfahren erforderlich, wenn Schutzgruppen srdf/a-Gerätegruppen haben (6733998)

Problembeschreibung: Die Sun Cluster Geographic Edition-Software lässt den Befehl geopg switchover nicht zu, wenn eine in einer Schutzgruppe enthaltene Gerätegruppe eine Symmetrix Remote Data Facility/Asynchronous (SRDF/A)-Gerätegruppe ist.

 Das in der Sun-Dokumentation enthaltene Verfahren funktioniert nicht für SRDF/A-Konfigurationen, da es den Befehl geopg switchover enthält.

Umgehung: Bevor Sie das Failback-Switchover-Verfahren durchführen, ändern Sie vorübergehend den Modus der Gerätegruppe von asynchron auf synchron, indem Sie folgenden Befehl ausgeben:

 # symrdf -g device-group set mode sync

 Wenn Sie das Failback-Switchover-Verfahren abgeschlossen haben, setzen Sie den Modus wieder auf asynchron zurück, indem Sie folgenden Befehl ausgeben:

 # symrdf -g device-group set mode async

clnode clear -F entfernt keine knotenbezogenen Informationen von clzc (6738437)

Problembeschreibung: Der Befehl clnode clear -F löscht den angegebenen Knoten aus einer Cluster-Konfiguration, aber nicht die zonenbezogenen Informationen auf dem zu löschenden Knoten.

Umgehung: Derzeit ist keine Umgehung verfügbar.

"clzc show -v" sollte nicht das verschlüsselte Root-Passwort für den Nicht-Root-Benutzer anzeigen. (6740154)

Problembeschreibung: Die Ausgabe des Befehls clzc show -v zeigt das verschlüsselte Root-Passwort für alle Benutzer an.

Umgehung: Derzeit ist keine Umgehung verfügbar.

scsi-Schlüssel werden bereinigt, selbst wenn der Fencing-Typ auf nofencing-noscrub eingestellt ist (6746190)

Problembeschreibung: Selbst wenn Sie den Fencing-Algorithmus für alle gemeinsam genutzten Geräte mit dem Befehl cldevice set -p oder cluster set -p auf nofencing-noscrub einstellen, bereinigt die Sun Cluster-Software trotzdem die Schlüssel für das gemeinsam genutzte Gerät.

Umgehung: Derzeit ist keine Umgehung verfügbar.

SVM-Root mit snv_97 auf Sparc führt zu Systemabsturz mit Threads in ndi_devi_enter (6744223)

Problembeschreibung: Dieser Fehler im Solaris (Nevada)-Kernel verhindert den ordnungsgemäßen Betrieb von Solaris Cluster Express 9/08.

Umgehung: Derzeit ist keine Umgehung verfügbar.

Booten von Zonen-Cluster auf Nevada nicht möglich (6746191)

Problembeschreibung: Bei Clustern, die sich aus Zonen des Typs cluster und native zusammensetzen, ist das Booten auf dem Solaris 10 Betriebssystem "Nevada" aufgrund von Inkonsistenzen mit Zugriffsrechten in der Zonen-Brand-Definition nicht möglich.

Umgehung: Derzeit ist keine Umgehung verfügbar.

clzc install schlägt beim Hinzufügen von Knoten zu einem bestehenden Zonen-Cluster fehl (6740700)

Problembeschreibung: Bei dem Versuch, einen neuen Zonen-Cluster mit dem Befehl clzonecluster install auf einem Knoten zu installieren, auf dem bereits ein Zonen-Cluster installiert ist, schlägt der Befehl fehl, und der neue Zonen-Cluster wird nicht installiert.

Umgehung: Derzeit ist keine Umgehung verfügbar.

[Vermillion]Java-Control-Panel enthält in vereinfachtem Chinesisch Quadratzeichen (6744135)

Problembeschreibung: Meldungen des Java Enterprise System (JES)-Installationsprogramms und anderen Java-Anwendungen werden im Gebietsschema für vereinfachtes Chinesisch nicht korrekt angezeigt. Aufgrund eines Schriftartproblems werden Meldungszeichenfolgen in Java-Anwendungen als Quadrate anstelle von fehlenden Schriftarten angezeigt.

Umgehung: Erstellen Sie folgenden symbolischen Link:

# cd /usr/openwin/lib/locale/zh_CN.GB18030/X11/fonts/TrueType
 # ln -s ./fangsongti.ttf songti.ttf
