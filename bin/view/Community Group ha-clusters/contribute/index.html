Contribute to Open High Availability Cluster

We welcome contributions from all members of the High Availability Clusters community! You can contribute by participating in discussions, finding bugs, fixing bugs, contributing to projects, starting your own project, and other activities.

  • First, read the general OpenSolaris contribution instructions.
  • Browse the Open HA Cluster source code.
  • Hang out in the Open HA Clusters discussions area about the code, bugs, requests for enhancement (RFEs), and other topics. You can also subscribe to the ha-clusters discussion group. When you subscribe, messages that members post to the discussion group area are sent to the electronic mail address that you specify.

Related Communities

  • The CTI for TET test framework is hosted by the testing community.
  • Open High Availability Cluster documentation is hosted by the documentation community. See the Open High Availability Cluster documentation participation page for how to contribute to Open High Availability Cluster documentation.

Specific Instructions

Fixing a Bug in the Software and in the Documentation

  1. Make sure you meet the following prerequisites:
    1* Know at least the basics about Solaris
    1* Know at least the basics about Solaris Cluster
    1* If the bug (called a change request, or CR by Sun) involves a data service, know at least the basics about the application for which the data service is developed
    1* Have access to at least a single-node cluster or a multi-node cluster on which you can develop and test. You can also run a single-node cluster within a virtualization environment like VirtualBox, VMWARE Workstation, Parallels, or similar software. You must be able to reproduce the problem that is described in the description of the bug. There is also a white paper which describes an example configuration to run a build system and a two-node development cluster environment on a single system, using VirtualBox, Open HA Cluster and OpenSolaris 2009.06.
  2. Find a bug that you want to fix. The bug can be a bug in the software, a bug in the documentation, or both. If you are not familiar with the cluster code base, search for bugs with the oss-bite-size keyword. Otherwise, look through all the cluster bugs, category suncluster/suncluster.
  3. Send an electronic mail message to request-sponsor@opensolaris.org. Note in the subject line that the bug in question is an Open High Availability Cluster bug. Provide the necessary information as described within the general OpenSolaris contribution instructions.
    A sponsor is assigned to you.
  4. Work with your sponsor on the bug.

Contributing to a Project

  1. Browse the list of active projects.
  2. Start participating in the project discussion list or send an electronic mail message to one of the project leads that asks how to contribute.

Starting a Project

  1. If you already have a project in mind, follow the instructions for proposing a project. Discuss your idea with the High Availability Clusters community leaders.
  2. If you want ideas about useful projects, see the Open High Availability Cluster wish list.

Translations

 Open High Availability Cluster translation projects are hosted by the translation community. See the Open High Availability Cluster Translation Guide for how to contribute to Open High Availability Cluster translation projects.
