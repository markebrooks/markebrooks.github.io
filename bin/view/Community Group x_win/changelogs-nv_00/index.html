Nevada Builds 6-9

X Change Logs: Nevada Builds 6-9

These changes went into the X Consolidation in Solaris Nevada Builds 6-9:

(The X consolidation did not start Nevada specific builds until Build 6.   Nevada builds 1-5 contained the X consolidation packages from the Solaris 10 builds they overlapped with.)

Build 06

  • 6205600 Xsun/sparc can't start with combo of m64/ffb and vfb.
  • 6199767 Xsun crashes when remote login into cinnabar linux
  • 6202808 Xsun version for NV release should be changed to 6.6.3
  • 6209164 Missing dependencies in X packages
  • 6209243 s10_73 sample locale testing get segmentation faults
  • 6207299 [Xorg bug \#1985] sync to 12/08/04 version of Nvidia driver from Mark Vojkovich
  • 6210823 Update from Xorg 6.8.0 to 6.8.2rc1
  • 6211538 Xorg tree updates for Nevada release

Build 07

  • 6213545 xorgconfig presents inconsistent keyboard selections
  • 6213860 [Xorg bug 1488] Xorg 6.8.0 & 6.8.1 segfault when using i810 drivers
  • 6217303 Xorg version info in log should be better
  • 6216976 Xsun tree build failures on nevada

Build 08

  • 5016163 \*accessx\* dumps core on events for buttons 4 and 5
  • 6178449 [cinnabar][xinerama] corrupt icons when operating in depth 24 on gfxp graphics adapters
  • 4921230 sv4Lib.tmpl contains tabs instead of spaces, causing make syntax errors
  • 4400050 [X.org defect \#8331] XtOpenApp and XtAppInit man pages have a bug
  • 6219947 Compose LED does not work in XKB mode
  • 4483703 ras2ps.1 should be in CDE man pages, not X
  • 6205323 fc-cache error message during install of CD2
  • 6219932 shift key breaks compose sequence when using XKB in C locale
  • 6198604 Xorg support for mice that use absolute coordinates
  • 6221637 [Xorg bug \#2380] Jan. 2005 nv chipset update

Build 09

  • 6227475 Some Asian & European keyboards don't work in Xorg
