Nevada Builds 170-175

X Change Logs: Nevada Builds 170-175

These changes went into the X Consolidation in Solaris Nevada Builds 170-175:

Build 170

  • 6979433 several X packages missing dependencies on ogl-select package
  • 7024876 Harden shell scripts in X
  • 7034214 X-redistributable should become X-all and use depend type=group
  • 7038014 Xorg Xserver should support -novtswitch option needed to solve drop-to-console issue when VT is used
  • 7051970 xrdb -merge generates spurious redefine warnings
  • 7053439 Xorg crash on usb mouse hotplug
  • 7054054 multiple X servers with NVIDIA cards doesn't work
  • 7059188 simplify and mogrify X meta-package synthesis

Build 171

  • 7053010 Xvfb/Xephyr fails to start as normal user
  • 7058808 Running 'fc-match --all' core dumps when no fonts are installed
  • 7059382 Upgrade FreeType from 2.4.4 to 2.4.5
  • 7061971 X builds shouldn't re-run pkglint when processed manifests haven't changed
  • 7062962 X pkg attribute name changes underscored by desire for consistency
  • 7066497 Xserver script shouldn't use readlink builtin
  • 7067904 Update deja-vu fonts to version 2.33

Build 172

  • 7035791 scanpci/libpciaccess should report 64 bit registers
  • 7041659 X packages should not deliver dir actions for directories defined elsewhere
  • 7069938 Stop delivering 75dpi & 100dpi iso8859-16 font dirs sans fonts
  • 7070321 Mouse protocol "Auto" does not work in build 170
  • 7071400 X gate needs to workaround -xregs=no%appl bug 7030022 in Studio 12.1 compilers
  • 7071917 Problem with xserver/font
  • 7073286 freeglut libraries not built with required flags like -xregs=no%appl
  • 7073739 Mass minor bugfix update to X.Org upstream release [July 2011 edition]

Build 173

  • 7034954 X packages have incorrectly multifaceted actions
  • 7043393 various parts of the X gate have compilation errors in debug mode
  • 7053074 [snv_166] xorg cores on launching gnome-panel-screenshot in labeled workspace
  • 7070538 Issues with korean input in notepad application
  • 7072633 Non-root users cannot start Xorg or Xvnc in snv_171
  • 7072864 X needs to implement the 'unlock' facet feature
  • 7073616 X package build versions: six dots are better than one
  • 7074024 Problem with xserver/font
  • 7074655 freeglut package includes doc files it shouldn't
  • 7076958 X properties created by the X server need to be marked as public information

Build 174

  • 7080479 X binaries change too much on rebuilds
  • 7081325 X builds should run check_rtime to ensure binaries are linked well
  • 7081442 Xorg mdb modules can't load
  • 7083886 Xfuncproto.h requires C99/gcc variable argument macros
  • 7084696 xdm default $PATH should stop including obsolete directories

Build 175

  • 6489124 docs needed for Xserver dtrace provider
  • 6932477 several S11 X11 man pages should remove Xsun references
  • 7083885 Excess noise in /var/log/Xorg.0.log when TX is enabled
  • 7086618 tigervnc (vncviewer) package should not depend on xserver packages
  • 7086632 X builds need to workaround pkgdepend bug 18884
  • 7086641 X package manifests should move to pkgfmt version 2
  • 7086762 oclock core dumps
  • 7088495 another build race in open-src/lib/DPS
  • 7088521 fbconfig may use or write to files it shouldn't
  • 7089465 x11/server/xorg/driver/xorg-efb is misnamed
  • 7094318 xemacs fails to start with "Error: Unresolved inheritance operation"
  • 7091272 Missing reference of .so in /usr/man/man3xi man pages
