Nevada Builds 140-149

X Change Logs: Nevada Builds 140-149

These changes went into the X Consolidation in Solaris Nevada Builds 140-149:

Build 140

  • 6942816 Console user update changes in Xorg
  • 6948254 Mass update of X gate sources to Oracle copyright format
  • 6948814 xrx will not build with Firefox 3.6 headers
  • 6948630 getifaddrs in 137 breaks Xvnc build
  • 6949653 Mass minor update to X.Org upstream release [April 2010 edition]
  • 6950022 X man page cleanups

Build 141

  • 6950900 Typing capslock twice in succession kills Xvnc
  • 6902956 Fontconfig version 2.8.0 [PSARC/2010/162]
  • 6949755 keyboard accessibility, mouse keys are unusable
  • 6954535 Update Xorg from 1.7.6 to 1.7.7
  • 6917536 MAXFILES is too small in makedepend

Build 142

  • 6949754 Xorg crashes when the magnifier is enabled at gdm login greeter window
  • 6950311 Xserver doesn't start on the Lenovo or Sony Vaio laptops with ironlake graphics
  • 6955153 RPATH and RUNPATH violations in xorg package
  • 6761802 RPATH/RUNPATH violation in SUNWxorg-graphics-ddx
  • 6959132 upgrade-X should allow installation on IPS installed systems
  • 6949184 Sony VAIO + GeForce 330M panic mapping free page
  • 6952880 TwinView display flicker on b139

Build 143

  • 6960017 Upgrade X client libraries to get xcb/thread-safety fixes
  • 6962316 X proto area should more closely match packages
  • 6962789 Fix a number of library runpath and unneededed linking issues in X gate

Build 144

  • 6900574 VNC - JAVA error when access VNC 5802 port from web
  • 6941932 X should be able to build IPS packages natively
  • 6963229 build race conditions and other makefile issues
  • 6964006 X package boundaries are all wrong
  • 6965575 Section 3X* man pages can't be found since move to /usr/share/man
  • 6966504 lndir [PSARC/2010/219]
  • 6967081 X package names need adjusting
  • 6963906 NVIDIA graphics driver support for OpenGL 4.0 [PSARC/2010/133]

Build 145

  • 6968523 developer/opensolaris/X can/should be part of X-redistributable
  • 6939384 X delivered SMF manifests should be relocated to /lib/svc/manifest
  • 6969277 Upgrade to TigerVNC 1.0.1
  • 6969142 Some directories have incorrect group ownership in new X delivery

Build 146

  • 6527729 Change xhost(1) man page to help with Secure by Default transition
  • 6929715 Xutf8TextListToTextProperty does not work for Korean text
  • 6970640 Update pixman to 0.18.2 [PSARC/2010/282]
  • 6971649 setxkbmap command dumps core when -device is specified without argument
  • 6972032 Java Application can not display Japanese Characters correctly in snv_144.
  • 6972645 var/log has incorrect group ownership in new X delivery 

Build 147

  • 6941479 Some new locale entries are missing in locale.dir and compose.dir
  • 6950572 X mouse driver crashes Xorg during unplug-plug the usb mouse device
  • 6972736 broken links to libXfont.so and libXres.so from package
  • 6974237 Support for ATI Technologies Inc Rage XL broken in build 144
  • 6974339 CVE-2010-1797 stack overflow in freetype 2.4.1 or older releases.
  • 6975036 Update freetype to 2.4.1
  • 6975153 libfreetype.so.6 does not contain correct mcs version info
  • 6975430 X copyright/license notice cleanup
  • 6977519 Mass minor update to X upstream releases [August 2010 edition]

Build 148

  • 6980366 Update freetype to 2.4.2
  • 6980773 X gate needs branding options
  • 6979204 updated gpl disclaimer text required
  • 6978644 More X license action updates

Build 149

  • 6982174 Restrictive firewalls hurt X build ; need to force use of local docbook dtd's
  • 6984028 bitstream-vera pkg has forgotten its legacy action name
  • 6984085 xlock build fails on snv_147: "life.c", line 444: identifier redeclared: kill
  • 6982494 temporary workaround for onu configuration issue needed for X gate
  • 6984035 more parallel build fixes for X gate
